Native vs. Web

Why Native? Why Web? Why must this be so confusing?

Fanboys argue about which platform is better or winning. Users just want the application to work on their device.
What is the most common platform we get requests for applications on?

iPad

But that's not the only platform they want to support.
While the most requests are for Android and iOS, you may still run in to some of these:
But then again, we haven't had a government client that has requested these.
Ok, so the name of your presentation is Native vs. Web. What gives? What's the best way to develop applications?

The safest way to develop applications is utilizing native SDKs.

Access to advanced APIs such as the camera and files system.

Speed

Developer Control

Ok, so why wouldn't I build all my mobile applications using native SDKs?
Because its expensive. Developers have to learn multiple languages and be familiar with multiple SDKs.
Alright, now is the time when we say "HTML5, CSS3 and JavaScript to the rescue!"
After all, we can do great things with CSS3 Transformations and in some cases they are graphically excellerated!
JavaScript libraries such as JQuery Mobile and Backbone.js have made building slick mobile interfaces and handling change events easy.
Third party tools like Phone Gap build JavaScript to Native bridges to allow advanced functionality.
Web developers are plentiful and easy to find!
Advanced CSS3 transitions are not fully supported across all devices. Even if they are supported, they may not be accelerated.
JavaScript multi-touch events aren't fully supported across devices.
Ok. You've painted us in to a corner here. What do we do? Here are your steps for success:
Know your requirements, and design for the mobile platform.
Use metaphors to the real world to drive interactions.
Once you've completed the design, choose the development stack.
Choose Native development when developing games or applications that require a platform specific API.
Choose HTML/CSS3/JavaScript for simpler applications that must be deployed cross platform.
